1. Before You Troubleshoot

Before diving into a specific issue area, complete the following pre-checks to confirm the platform is in a known state. Many issues trace back to configuration drift or a failed upgrade that left the appliance in a degraded condition.

1

Verify appliance health

Log in to the Actifio Desktop or AGM (Actifio Global Manager) and check the Appliance Health dashboard. Confirm all services show Running status and that no active alerts are present.

2

Check the system event log

Navigate to Monitor > Events and filter by Error and Warning severity for the past 24 hours. Note any repeating event IDs — recurring events often indicate a systemic root cause rather than a transient failure.

3

Confirm licence validity

Go to Manage > Appliances > Licence. Ensure the licence is not expired and that the consumed capacity is within the licensed limit. Capacity overruns cause new backup jobs to queue indefinitely.

4

Collect diagnostic bundle (if escalating)

If you intend to contact Actifio Support, generate a diagnostic bundle before making any changes. Navigate to Manage > Appliances, select the appliance, and click Collect Diagnostics. Attach the bundle to your support case.

2. Backup Job Failures

Backup job failures are the most common category of issue. Each failure generates an event with a specific error code. Use the Error Code Reference to identify the category, then apply the matching resolution below.

Backup job remains in Queued state for more than 30 minutes

Cause No available resource pool slots. The appliance is at maximum concurrent job capacity, or the SLA window has shifted so that multiple policies triggered simultaneously.
Fix Navigate to Manage > Appliances > Resource Pools and check the current slot utilisation. If utilisation is at 100%, either wait for running jobs to complete or increase the pool size. To stagger concurrent triggers, edit the SLA template and offset the start times by 5–10 minutes.
CLI udsinfo lsresourcepool — lists all resource pools and current utilisation.

Backup job fails with Connection refused or Host unreachable error

Cause The Actifio connector on the source host is not running, or the host firewall is blocking the required ports (TCP 5106, 26257).
Fix On the source host, verify the Actifio UDS Agent service is running: systemctl status udsagent. If stopped, start it: systemctl start udsagent. Confirm firewall rules allow inbound TCP 5106 and 26257 from the Actifio appliance IP.
CLI udsinfo lshost -filtervalue hostname=<hostname> — confirms the registered host status and last successful check-in time.

Database backup fails immediately after a source DB failover or restart

Cause Actifio retains a stale application discovery record pointing to the old DB instance. After a failover, the cached application state does not reflect the new primary.
Fix Force a re-discovery of the application: navigate to Manage > Applications, select the affected database, and click Discover Now. Wait for discovery to complete, then retry the backup job.
CLI udstask discovercluster -host <hostname>

Backup completes but SLA compliance shows Missed in the dashboard

Cause The backup completed outside the allowed SLA window. This is common when a preceding long-running job delayed the current job's start time past the window close.
Fix Review the SLA template and extend the allowed window, or enable Catch-up Jobs on the policy so Actifio schedules a compensating run immediately after the window closes. If SLA misses are frequent, analyse the job duration trend in Monitor > Reports > Job Summary and adjust the window accordingly.
Important
Do not delete and re-add an application to resolve a recurring backup failure. This removes all existing image points and resets the protection history. Always use Discover Now or re-configure the SLA as the first remediation step.

3. Replication and StreamSnap Issues

Replication transfers backup images to a remote Actifio appliance. StreamSnap is the near-synchronous replication option for RPO-sensitive workloads. The issues below cover both technologies.

Replication lag exceeds the configured RPO threshold and generates a compliance alert

Cause Insufficient WAN bandwidth between source and target appliances, or a temporary network congestion spike that caused the replication queue to back up.
Fix Check the replication throughput in Monitor > Replication. If throughput is consistently below the required rate, increase the replication bandwidth allocation in the network throttle settings. For temporary lag, allow the queue to drain before raising a support case.
CLI udsinfo lsreplicationlag -targetappliance <appliancename>

StreamSnap jobs fail with Remote appliance not reachable

Cause The inter-appliance trust relationship has expired or was disrupted after a firmware update on either side.
Fix Re-establish the trust relationship: in AGM, navigate to Manage > Appliances, select the target appliance, and click Re-Trust. If AGM is not in use, run udstask addcluster -target <target_ip> from the source appliance CLI.
CLI udsinfo lscluster — verify the trust state of all paired appliances.
Critical
Do not remove a replication policy while a StreamSnap job is in progress. Doing so can leave the target appliance with a partial and unusable image set. Always wait for the current job to complete or explicitly cancel it before modifying the policy.

4. Mount and Restore Issues

Actifio's instant virtualisation (mount) and restore operations are both served from backup images. The troubleshooting steps differ depending on whether you are mounting a virtual copy or initiating a full restore.

Instant mount succeeds but the mounted volume is not visible on the target host

Cause The iSCSI initiator on the target host has not logged in to the Actifio target portal, or the multipath daemon is presenting the device under a non-standard path.
Fix On the target host, run iscsiadm -m session to confirm the initiator session is active. If not, log in: iscsiadm -m node --login. Then rescan: rescan-scsi-bus.sh or echo 1 > /sys/class/scsi_host/host*/scan. Verify the device appears with lsblk.

Database restore fails with Insufficient staging disk space

Cause The staging area on the target host does not have enough free space to accommodate the uncompressed restore data. Actifio estimates staging space from the source image size, not the compressed on-appliance size.
Fix Free at least 120% of the source database size on the staging path before retrying the restore. Alternatively, redirect the staging path to a volume with sufficient space via the restore wizard's Advanced Options > Staging Path setting.

5. Appliance and Network Connectivity

Actifio Desktop cannot connect to the appliance — login page times out

Cause The Actifio web service (port 443) is not responding, or the management network interface is down.
Fix From a host on the management network, ping the appliance management IP. If ping fails, check the physical or virtual NIC connected to the management network. If ping succeeds but HTTPS times out, SSH to the appliance and run service actifio status to check the web service.
CLI ssh admin@<appliance_ip> then service actifio restart if the service is stopped.

AGM shows appliance status as Unreachable even though the appliance is online

Cause The AGM-to-appliance heartbeat uses port 26257. If a firewall rule change or network ACL update has blocked this port, AGM marks the appliance as unreachable even while it continues to protect data independently.
Fix From the AGM server, confirm TCP 26257 is open to the appliance IP: nc -zv <appliance_ip> 26257. Restore the firewall rule if blocked. After connectivity is restored, AGM will automatically reconcile within 5 minutes — no manual resync is required.

6. Performance and Resource Issues

Backup throughput drops significantly compared to previous runs of the same job

Cause Storage pool fragmentation on the appliance, high concurrency competing for the same back-end pool, or the source host experiencing I/O saturation during business hours.
Fix Run the storage pool defragmentation task: Manage > Storage > Defragment. Schedule high-throughput backup jobs to run outside business hours using SLA window controls. Review the source host I/O metrics during the backup window to confirm it is not the bottleneck.
CLI udsinfo lsdiskpoolstatus — shows fragmentation level and available capacity per pool.

Appliance CPU utilisation stays above 90% for extended periods

Cause Too many concurrent deduplication and compression threads are running simultaneously. This is common after a catchup burst following a maintenance window.
Fix Temporarily reduce the maximum concurrent job count in Manage > Appliances > Resource Pools to allow the appliance CPU to recover. Increase the limit gradually after CPU drops below 70%.
CLI udsinfo lsjobhistory -filtervalue status=running — lists all currently running jobs and their start times.
Best Practice
Enable Workload Balancing in the SLA policy settings to allow Actifio to automatically stagger concurrent jobs based on available appliance resources. This prevents peak-hour CPU spikes without requiring manual intervention.

7. Error Code Reference

The following error codes appear in Monitor > Events and in job failure notifications. Use the Severity column to prioritise remediation.

Error Code Severity Description Resolution
AGM-0001 Critical Appliance unreachable from AGM See Section 5 — check TCP 26257 connectivity
JOB-1042 Critical Backup job failed — host agent not responding Restart the UDS agent on the source host; verify firewall allows TCP 5106
JOB-1101 Warning Backup completed outside SLA window Enable catch-up jobs or extend the SLA window; see Section 2
REP-2010 Critical Replication target trust relationship invalid Re-trust the remote appliance in AGM; see Section 3
REP-2055 Warning Replication queue depth exceeds threshold Increase WAN bandwidth allocation or reduce concurrent replications
STG-3001 Critical Storage pool capacity above 90% Expire old images or add storage capacity to the pool immediately
STG-3040 Warning Disk pool fragmentation above 25% Run defragmentation task from Manage > Storage
MNT-4012 Warning Mounted volume not visible on target host Rescan iSCSI initiator on target; see Section 4
LIC-5001 Info Licence capacity at 80% — approaching limit Review image expiry policies or request a licence expansion
LIC-5010 Critical Licence capacity exceeded — new jobs will not start Immediately expire old images or contact Actifio Support for an emergency licence
Tip
Filter the Events log by error code prefix (for example, REP-) to isolate all replication-related events for a given time range. This is faster than scrolling through mixed event types during an incident.
