1. How SmartCode Works

The VWO SmartCode is a JavaScript snippet that connects your website to VWO's servers. When a visitor loads a page, the SmartCode determines which active experiments apply, fetches the relevant variation data from VWO's CDN, and modifies the page DOM before the visitor sees the content.

SmartCode Execution Flow

The SmartCode is installed once and works across all pages. You do not need to reinstall it for each experiment. VWO's CDN serves only the experiment data relevant to the current page and visitor, keeping the payload minimal.

2. Before You Start

Confirm the following before installing SmartCode:

  • You have a VWO account. If not, start a free trial.
  • You can add JavaScript to your website's <head> tag, either directly or through a developer.
  • You have registered your domain in VWO under Configurations > Websites and Apps.
Do Not Use Tag Managers
VWO strongly recommends installing SmartCode directly in your HTML, not through Google Tag Manager or other tag managers. Tag managers load scripts asynchronously and cannot guarantee execution order, which causes page flicker and timeout issues with A/B tests.

3. Choose Your SmartCode Type

VWO offers two SmartCode variants. Choose based on your performance requirements and site architecture.

SmartCode Versions
VWO currently offers SmartCode 2.1 and 2.2. Version 2.2 includes performance improvements for SSG frameworks (Next.js, Nuxt, Gatsby) and better tag manager compatibility. Check Configurations > Websites and Apps > Code for your current version. Upgrading is optional but recommended for SSG-based sites.

4. Install SmartCode (Manual Method)

1

Navigate to your SmartCode

Go to Configurations > Websites and Apps. Click on your domain. Select the Code tab.

2

Select Async or Sync

VWO shows the Asynchronous version by default. To switch: click the gear icon (⚙) at the top-right of the SmartCode section, select Synchronous, then click Save.

3

Copy the SmartCode

Click Copy. If you need a developer to install it, click the Share icon to send a pre-written email with the snippet.

4

Paste into your website's <head> tag

Sign in to your website's admin panel and paste the snippet between the <head> and </head> tags. If your site uses template-based headers, add it to the header template so it applies sitewide.

HTML <head> <!-- Your existing meta tags, CSS, etc. --> <!-- VWO SmartCode: paste BEFORE the closing </head> tag --> <script type="text/javascript" id="vwoCode"> // Your unique SmartCode from VWO dashboard // Contains your Account ID for server communication </script> </head>
5

Verify the installation

See Section 6: Verify Your Installation below.

One-time setup
SmartCode installation is a one-time task. Once installed, you can create unlimited experiments without touching the code again.

Alternative: Share SmartCode with a Developer

If you do not have access to your website's code, click the Share icon at the top-right of the SmartCode section in VWO. This generates a pre-written email with the code snippet and installation instructions that you can send directly to your developer.

Alternative: Auto-Install via Plugins

VWO provides plugins for WordPress, Drupal, Joomla, Wix, and other platforms that automatically add SmartCode to all pages without manual code editing. See Extensions and Plugins in VWO SmartCode for the full list.

Cross-Domain Campaigns: Browser Cookie Settings

If you run experiments across multiple domains, the latest versions of Chrome, Safari, and Firefox block third-party cookies by default. To enable cross-domain tracking during your testing:

  • Chrome: Settings > Privacy and security > Third-party cookies > Allow third-party cookies
  • Safari: Settings > Privacy > disable "Prevent cross-site tracking"
  • Firefox: Settings > Privacy and Security > Custom > disable "Cookies" and "Tracking Content"
Cookie Consent
If your privacy policy requires visitor consent before running VWO, see Cookie Consent and VWO's Adherence for implementation approaches that delay SmartCode execution until consent is granted.

5. Framework-Specific Installation

If your site uses a JavaScript framework or CMS, follow the specific method for your stack. Click your framework below.

N
Next.js
Page Router & App Router
Install the NPM package: npm install vwo-smartcode-nextjs.

Page Router: Import and add VWOScript in _document.js inside <Head>.
App Router: Add VWOScript to app/layout.tsx inside the <head> element.

Do not add SmartCode manually — always use the NPM package for Next.js.
N
Nuxt.js
nuxt.config configuration
Open nuxt.config.

In head.link, add a preconnect entry for https://dev.visualwebsiteoptimizer.com.

In head.script, add your SmartCode snippet using innerHTML to prevent escaping.
G
Gatsby
Version 4.19.0 and later
Create a component that uses dangerouslySetInnerHTML with your SmartCode (script content only, without the wrapping <script> tag) in the __html key.

Add this component to your page layout. For Gatsby versions earlier than 4.19.0, use React Helmet instead.
R
React Helmet
Asynchronous SmartCode only
Wrap your SmartCode in template literal backticks (`…`) and brackets inside the <Helmet> tag.

Important: Use Asynchronous SmartCode only. Synchronous SmartCode conflicts with React Helmet's asynchronous script injection and will not execute correctly.
H
Shopify Hydrogen
V1 and V2
V1: Paste SmartCode directly in index.html inside the <head> tag.

V2: Define a getVWOAsyncScript() function in app/root.jsx that returns your SmartCode via the __html property. Render it using dangerouslySetInnerHTML at the very start of <head>.
W
WordPress
Plugin — no code editing needed
Install the VWO plugin from the WordPress Plugin Directory.

Once activated, the plugin automatically adds SmartCode to every page on your site. No manual code editing is needed. The plugin also provides a SmartCode verification tool directly from the WordPress admin panel.
CF
Cloudflare
Rocket Loader conflict fix
Cloudflare's Rocket Loader defers all JavaScript, including SmartCode, which breaks execution order and causes flicker.

To prevent this, add data-cfasync="false" to your SmartCode <script> tag. This tells Rocket Loader to skip this script and let it execute synchronously.
+
Other CMS & Platforms
Drupal · Joomla · Wix · and more
VWO provides dedicated plugins for Drupal, Joomla, Wix, and other platforms. Each plugin automatically handles SmartCode placement — no manual <head> editing required.

See Extensions and Plugins in VWO SmartCode for the full list with installation links.
React Helmet Users
Do not use the Synchronous SmartCode with React Helmet. React Helmet injects scripts asynchronously, which conflicts with synchronous execution. Use Asynchronous SmartCode only.

6. Verify Your Installation

After installing SmartCode, verify it is working correctly using one of these methods:

Method 1: VWO SmartCode Checker (Recommended)

1

Open the SmartCode Checker

Go to Configurations > Websites and Apps > your domain > Check if SmartCode is installed properly on your URL.

2

Enter your URL and click Check

VWO opens your page in a new tab, waits up to 30 seconds for the SmartCode to execute, then reports the result.

Expected Results

ResultWhat It MeansAction
Correctly installedSmartCode is running and connected to your account.You're done. Start creating experiments.
Different account IDA SmartCode from a different VWO account is installed on this URL.Replace the old SmartCode with your current account's snippet.
Could not verify (30s timeout)SmartCode is missing, not executing, or being blocked.See Troubleshooting below.
URL redirectedYour URL redirected to a different page. VWO checked the redirected URL.Verify SmartCode is on the final destination URL.

Method 2: Browser Developer Tools

Open Chrome DevTools (F12), go to the Network tab, and filter by dev. Look for these network calls:

  • Asynchronous SmartCode: look for j.php
  • Synchronous SmartCode: look for gv.gif and {your_account_id}.js

If these requests appear and return 200 OK, your SmartCode is working.

7. Troubleshooting

Common issues and how to resolve them.

Page flicker: visitors briefly see the original page before the variation loads
CauseSmartCode is loading too late (placed at the bottom of <head> or loaded via a tag manager).
FixMove SmartCode as high as possible in the <head> tag. Do not use Google Tag Manager to load VWO. If flicker persists, switch to Synchronous SmartCode.
"SmartCode not installed" warning in VWO dashboard
CauseNo visitor has triggered the SmartCode on the checked page yet, or the code is not executing.
FixVisit the page yourself to trigger the first execution. Refresh the VWO dashboard after a few minutes. This warning does not affect live test data if the code is present in the HTML.
SmartCode blocked in incognito mode (Firefox, Safari)
CauseDefault privacy settings in Firefox and Safari block cross-site tracking in private browsing.
FixFor Safari: Preferences > Privacy > disable "Prevent cross-site tracking." For Firefox: Preferences > Privacy and Security > Custom > disable "Cookies" and "Tracking Content." This affects your testing environment only, not your visitors' normal browsing.
Ad blocker (uBlock Origin) blocks VWO
CauseuBlock blocks network calls to dev.visualweboptimizer.com, preventing the Visual Editor, Heatmaps, and SmartCode Checker from working.
FixClick the uBlock icon > Settings (gear) > My Filters tab > add @@||*.visualwebsiteoptimizer.com^ > click Apply Changes.
SmartCode installed multiple times on the same page
CauseDuplicate installation (e.g., both manually and via a plugin) causes screen flicker, blank pages, and double-counting.
FixSearch your page source for "vwoCode" and ensure it appears only once. Remove any duplicates.
Tests not running on specific pages
CauseSmartCode is only installed on some pages (e.g., homepage but not product pages).
FixInstall SmartCode in your site's global header template so it loads on every page. For iframe content, install SmartCode on the iframe URLs separately.
Still stuck?
Use VWO's Preview Debugger: select your test, go to Review > Live Preview, and click Preview in a new tab. This confirms whether VWO can communicate with your page. If problems persist, contact support@vwo.com.
