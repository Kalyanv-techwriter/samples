Actifio Short RPO Configuration Guide

Configure Short RPO (Recovery Point Objective) policies in Actifio 10.0 to protect high-priority workloads with sub-hour recovery points and near-continuous data protection.

1. What Is Short RPO?

Concept
Recovery Point Objective (RPO) is the maximum acceptable age of a backup that can be used to recover from data loss. A lower RPO means more frequent backups and less potential data loss in the event of a failure. Actifio's Short RPO mode enables backup intervals as low as every 15 minutes — far below the standard 1-hour minimum — using a lightweight change-block-tracking mechanism to capture only the data that has changed since the previous image.

Standard Actifio SLA policies support backup frequencies of 1 hour to 24 hours. Short RPO extends this range downward, enabling frequencies of 15, 20, or 30 minutes. This is particularly valuable for workloads where even one hour of data loss is unacceptable — such as transactional databases, financial ledgers, and ERP systems running active business operations.

Short RPO images are taken as incremental-forever backups on top of a full baseline. Each incremental capture is lightweight and completes in seconds to minutes, depending on the change rate of the source data.

2. When to Use Short RPO

Short RPO is designed for specific workload categories. Review the table below before enabling it — the additional resource overhead is only justified for workloads with strict RPO requirements.

Criterion Standard RPO (≥1 hr) Short RPO (15–30 min)
RPO requirement 1 hour to 24 hours acceptable Less than 1 hour required
Workload type File shares, dev/test, archival Transactional databases, ERP, financial systems
Change rate on source Low to medium Medium to high (Short RPO is efficient regardless due to CBT)
Appliance resource impact Low Higher — more frequent scheduling threads
Storage pool consumption Lower Higher — more image points retained
Licence impact Standard licence covers Requires Short RPO licence entitlement
Supported application types All applications VMware VMs, Oracle, SQL Server, SAP HANA
Licence Requirement
Short RPO requires a specific licence entitlement. Verify that your Actifio licence includes Short RPO before proceeding. To check, navigate to Manage > Appliances > Licence and look for Short RPO in the feature list. If it is absent, contact your Actifio account representative.

3. Prerequisites

Verify all prerequisites before configuring a Short RPO policy. Applying Short RPO to an environment that does not meet these requirements will result in failed or queued jobs.

Pre-Configuration Checklist
Actifio appliance version 10.0 or later. Short RPO requires platform version 10.0. Earlier versions do not support sub-hourly scheduling.
Short RPO licence entitlement is active. Verified in Manage > Appliances > Licence.
UDS Agent version 10.0 or later installed on source host. Download the latest agent from the Actifio Support Portal and install it before applying the policy.
Change-block tracking (CBT) enabled on the source application. For VMware, CBT must be enabled on the VM. For Oracle and SQL Server, the Actifio agent enables this automatically on first backup.
Sufficient storage pool capacity. Short RPO retains more image points. Confirm the storage pool has at least 30% free capacity before enabling. Check in Manage > Storage Pools.
Resource pool has available slots for sub-hourly job frequency. Each 15-minute interval generates a job thread. Ensure the resource pool maximum is set high enough to avoid persistent queuing during peak windows.
A successful standard backup image already exists. Short RPO incrementals are built on top of a full baseline image. If no baseline exists, Actifio runs a full backup before beginning sub-hourly incrementals.

4. Configure the Short RPO SLA Template

Short RPO is configured within an SLA template. You can create a new template specifically for Short RPO workloads or modify an existing template to enable sub-hourly scheduling.

Recommendation
Create a dedicated SLA template for Short RPO rather than modifying an existing one. This makes it easy to identify which applications are protected under Short RPO and to audit the configuration independently from standard policies.
1

Open the SLA Architect

In the Actifio Desktop or AGM, navigate to SLA Architect from the left navigation panel.

2

Create a new SLA template

Click + New Template. Enter a descriptive name such as ShortRPO-15min-Production. Select the application type (for example, VM or Oracle) from the Application Type dropdown.

3

Add a Snapshot policy and enable Short RPO

Under the Snapshot policy section, click Add Policy. In the policy settings panel that appears, toggle Short RPO to Enabled. The scheduling frequency options will update to include sub-hourly intervals.

4

Set the Short RPO fields

Configure the fields below. All fields become visible once Short RPO is enabled.

Frequency
How often Actifio captures an incremental image. Options: 15 min, 20 min, 30 min. Choose 15 min only if the source workload change rate justifies it and resource pool slots are available.
Retention
How many Short RPO images to retain. Each image occupies storage pool space proportional to the source change rate. A typical setting is 4 hours of rolling images (16 images at 15-min frequency).
Start Window
The time range within which Short RPO jobs are allowed to run. For always-on protection, set this to 00:00 – 23:59 (full day). Restrict the window if the source system has maintenance periods where backup should pause.
On Demand Retention
How long manually triggered Short RPO images are retained. Set independently from the scheduled retention. Useful for capturing a known-good baseline before a deployment window.
Enforce SLA
When enabled, Actifio generates a compliance event if a scheduled interval is missed. Enable this for production workloads so SLA misses are visible in Monitor > Events.
5

Save the template

Click Save Template. The template is now available to assign to applications.

5. Apply the Policy to an Application

1

Locate the application

Navigate to Manage > Applications. Use the search bar to find the application by name or host. Confirm that the application shows a status of Discovered before assigning a policy. If not, click Discover Now and wait for the status to update.

2

Assign the SLA template

Right-click the application and select Manage SLA. In the Select SLA Template dropdown, choose the Short RPO template you created. Review the policy summary and click Apply.

3

Trigger the initial baseline backup

If no prior snapshot exists for this application, Actifio automatically schedules a full baseline backup before starting Short RPO incrementals. You can monitor the baseline job in Monitor > Jobs. The baseline may take significantly longer than subsequent incrementals depending on source data size.

4

Confirm the Short RPO schedule is active

After the baseline completes, navigate to Manage > Applications, select the application, and view the SLA Details tab. Confirm the next Short RPO job is scheduled and that the policy shows Short RPO: Enabled.

Important
Assigning a Short RPO policy to an application that already has a standard policy replaces the existing policy entirely. If the existing policy includes a separate dedup or replication policy, you must re-add those policies to the new Short RPO template before saving.

6. Verify the Configuration

After applying the policy, verify that Short RPO is operating correctly before considering the configuration complete.

Via the Actifio Desktop or AGM

  • Navigate to Monitor > Jobs and filter by the application name. Confirm that Short RPO incremental jobs are appearing at the configured interval (for example, every 15 minutes).
  • Check the job duration — Short RPO incrementals should complete in under 5 minutes for typical workloads. Longer durations indicate a high change rate or resource contention.
  • Navigate to Manage > Applications > Images for the application and confirm multiple image points exist within the past hour, spaced at the configured interval.

Via the CLI

udscli # List the most recent image points for the application udsinfo lsbackup -filtervalue appname=<app_name> \ -filtervalue jobclass=snapshot \ | head -20 # Confirm Short RPO is enabled on the SLA template udsinfo lssla -filtervalue slaname=<template_name> # View the scheduled next run time udsinfo lsjobschedule -filtervalue appname=<app_name>
Success Indicators
The configuration is working correctly when: (1) incremental job entries appear in the job list at the configured frequency, (2) each job completes with a Succeeded status, and (3) the image list for the application shows distinct image points spaced at the configured interval.

7. Best Practices and Considerations

Limit Short RPO to critical workloads

Short RPO increases the scheduling frequency on the appliance and consumes more storage pool capacity per application. Apply it selectively — only to workloads that genuinely require sub-hour RPO. For other applications, standard hourly policies are more resource-efficient.

Monitor storage pool growth

Short RPO retains more image points, which accelerates storage pool consumption. Set a pool utilisation alert at 75% in Manage > Storage Pools > Alerts so you have lead time to either expire older images or expand capacity before hitting the 90% threshold that triggers job blocking.

Align the SLA window with source system availability

If the source system undergoes scheduled maintenance (for example, nightly database statistics rebuilds or OS patching), configure the Short RPO window to exclude these periods. Attempting to capture incremental images during a database restart will result in job failures and SLA misses.

Test recovery at the configured RPO

After enabling Short RPO, perform a test mount of the most recent incremental image to confirm it is recoverable. A Short RPO policy that captures images but cannot mount them successfully does not actually reduce your effective RPO.
