Introducing Actifio GO

Get started with Actifio GO, the SaaS-based Virtual Data Pipeline. This guide walks you through access, the dashboard, your first backup, and key platform capabilities.

1What Is Actifio GO?

Actifio GO is the SaaS-delivered edition of the Actifio Virtual Data Pipeline (VDP). Unlike the on-premises appliance model, Actifio GO is hosted and managed by Actifio — you consume it as a service with no hardware to rack, no software to patch, and no infrastructure to maintain. Your backup jobs, images, and policies are managed through a web portal and a lightweight connector installed on your source hosts.

Actifio GO Architecture — How Data Flows
SaaS
No infrastructure to manage
Actifio operates the VDP engine, storage pools, and replication infrastructure. You manage only what you want to protect.
RPO
Flexible recovery objectives
Schedule backups from every 15 minutes (Short RPO) to daily. SLA templates enforce protection consistency automatically.
VDP
Virtual Data Pipeline
Incrementally-forever backups with deduplication and compression. First backup is a full image; subsequent backups capture only changed blocks.
MNT
Instant mount for recovery
Mount any image point as a virtual copy in minutes without restoring data. Ideal for rapid recovery testing, dev/test provisioning, and analytics.

2Prerequisites and Access

Actifio GO subscription activated. Your organisation admin receives a welcome email with the GO portal URL and initial admin credentials.
Browser: Chrome 90+, Firefox 88+, or Edge 90+. JavaScript must be enabled.
UDS Agent installed on each source host (Windows or Linux). Download link is available from the GO portal after login.
Network ports open: TCP 5106 and 26257 outbound from source hosts to the Actifio GO cloud endpoint.
Source host credentials with sufficient privileges: root or Administrator, or a service account with backup operator rights for the target application.
For VMware: vCenter credentials with read access to inventory and datastore. CBT (Changed Block Tracking) must be enabled on VMs to be protected.

3First Login and Initial Setup

Onboarding Flow — First 30 Minutes
1

Log in and set your password

Open the GO portal URL from your welcome email. Use the temporary password provided to log in. You will be prompted to set a permanent password on first access. Enable MFA under My Profile > Security before proceeding.

2

Download and install the UDS Agent

Navigate to Manage > Agents. Download the agent installer for your OS (Linux RPM/DEB or Windows MSI). Run the installer on each source host as root or Administrator. The agent registers automatically with your GO instance using the embedded account token.

3

Add and verify hosts

Go to Manage > Hosts > Add Host. Enter the hostname or IP address. GO attempts to connect via the agent. A green Reachable status confirms successful registration. Hosts showing Unreachable — check that TCP 5106 is open outbound.

4

Discover applications

Select a registered host and click Discover Apps. GO scans for supported database instances, VMs, and file systems and lists them in the Applications inventory. VMware VMs are discovered automatically once vCenter credentials are added under Manage > Credentials.

4Dashboard Walkthrough

The GO home dashboard gives you a real-time view of your data protection posture. The layout is divided into four primary zones.

GO Dashboard Layout
A — Summary Tiles
At-a-glance counts for protected apps, SLA compliance percentage, active alert count, and total data under protection. Click any tile to drill into that category.
B — Recent Jobs
Live feed of the last 20 job executions — sorted by start time with status, duration, and type. Click any row to view full job details and event log.
C — SLA Compliance by Tier
Compliance rate broken down by your defined SLA tiers (Gold, Silver, Bronze). Bars turn amber below 95% and red below 90% to flag which tier needs attention.
D — Active Alerts
Critical and warning events that require action — backup failures, capacity warnings, connectivity issues. Each alert links directly to the affected resource.

5Configure Your First Backup

Backup Configuration Flow
1

Select the application

Go to Applications in the top navigation. Locate your app in the inventory. If it is not listed, click Discover Now on the parent host. Right-click the application and select Manage SLA.

2

Assign an SLA template

From the Select SLA Template dropdown, choose a pre-built template (for example, Gold-Hourly or Silver-4hr). If no suitable template exists, create one under SLA > New Template before returning to this step. Click Apply.

3

Run the first on-demand backup

Right-click the application and select Backup Now. This initiates a full baseline image. The first backup will be significantly larger than subsequent incrementals — duration depends on the source data size and network throughput.

4

Monitor the job

Navigate to Jobs. Locate the running job — it will show the application name, job type (Snapshot), start time, and elapsed duration. A green Succeeded status confirms the first image is captured and available.

First Image Captured
Once the job completes, go to Applications > select the app > Images. You will see the first image point with its timestamp and size. This image is immediately available for instant mount or restore.

6SLA Policies and Protection Plans

An SLA template defines how often backups run, how long images are retained, and what window the scheduler is allowed to use. You attach one template to one or many applications.

GoldEvery 1 hour24 images (rolling 24 hrs)Production transactional databases, ERP, payroll

FieldWhat It Controls

Built-in Templates
GO ships with Gold, Silver, and Bronze templates ready to use. You can clone and customise any template under SLA > Templates > Clone. Do not edit the built-in templates directly — cloning preserves the originals as a reference baseline.

7Key Platform Features

Instant Mount

Mount any captured image as a virtual copy on a target host in minutes — without copying data. The mount is read-write capable and serves data directly from the GO image store. Use cases: rapid recovery testing, dev/test database provisioning, analytics on a production copy without impact to the live system.

Instant Mount Flow
CBT
Changed Block Tracking
After the full baseline, GO captures only changed blocks. Incremental jobs run in minutes, not hours. Storage consumption grows linearly with your change rate, not your total data size.
DD
Global Deduplication
Deduplication operates across all protected applications in your organisation. Identical blocks from different hosts are stored only once, significantly reducing storage footprint for environments with common OS and application binaries.
RPL
Cross-Region Replication
Replicate backup images to a second GO region for disaster recovery. Replication is policy-driven — configure it within the SLA template. Only incremental changes are transferred after the initial seed, minimising WAN bandwidth consumption.
RPT
Reports and Compliance
Pre-built reports under Reports cover SLA compliance, job history, storage consumption, and data reduction ratios. Schedule reports to be emailed as PDF or CSV. Use the compliance report as audit evidence for data protection SLAs.
Storage Pool Monitoring
GO manages cloud storage on your behalf but charges are based on your retention policy. Review the Storage Consumption report monthly and expire old images on applications where retention requirements have changed to avoid unnecessary cost.

8Next Steps

1
Test a recovery
Immediately after the first backup completes, mount the image on a test host and verify the data is accessible. Never assume a backup is usable until it has been tested.
2
Onboard remaining applications
Repeat the host registration and application discovery steps for all workloads that need protection. Bulk-assign SLA templates by multi-selecting applications in the inventory.
3
Set up DR replication
For applications with RTO/RPO requirements that span site failure, add a replication policy to the SLA template and configure a target GO region under Manage > Replication Targets.
4
Schedule compliance reports
Set up a weekly SLA Compliance report to be emailed to your team under Reports > Scheduled Reports. This closes the loop on whether your protection objectives are being met.
